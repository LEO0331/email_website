# Email Survey App - Local Storage Demo

A simple survey creation and management app that runs entirely in the browser using localStorage. Perfect for GitHub Pages deployment with zero backend dependencies.

## What's Changed

### ✅ Migrated From MongoDB
- Removed all MongoDB/Mongoose dependencies
- All data stored in browser's localStorage
- No server-side database needed

### ✅ Removed Authentication & Payments
- No Google OAuth required
- No Stripe payments
- No user accounts

### ✅ Simplified Backend
- Minimal Express server that only serves frontend files
- Works with GitHub Pages (optional frontend-only deployment)
- Can also run locally with `npm run dev`

## Getting Started

### 1. Update Client Code

Follow [MIGRATION_GUIDE.md](./MIGRATION_GUIDE.md) to update:
- `client/src/actions/index.js` → Use localStorage instead of API calls
- `client/src/reducers/surveyReducer.js` → Keep Redux for state management
- Optionally remove auth/payment components from `client/src/component/Header.js`

A complete example is provided in `client/src/actions/index.example.js`.

### 2. Install Dependencies

```bash
npm install
```

### 3. Run Locally

```bash
npm run dev
```

- Backend: http://localhost:5000
- Frontend: http://localhost:3000 (React dev server)
- All surveys are saved to browser localStorage

### 4. Test the App

1. Navigate to http://localhost:3000
2. Create a new survey (no login required)
3. Fill in title, subject, body, and recipient emails
4. Submit survey - it's saved to browser storage
5. View surveys on dashboard
6. Refresh page - surveys persist!
7. (Optional) Open DevTools → Application → Local Storage to see raw data

## Deployment to GitHub Pages

### Option A: GitHub Pages Only (Recommended for Demo)

1. **Build the frontend:**
   ```bash
   cd client
   npm run build
   cd ..
   ```

2. **Push to GitHub:**
   ```bash
   git add -A
   git commit -m "Deploy survey app to GitHub Pages"
   git push origin main
   ```

3. **Enable GitHub Pages:**
   - Go to repo Settings → Pages
   - Source: Deploy from a branch
   - Branch: `main`, folder: `client/build`
   - Visit your GitHub Pages URL

4. **Note:** static routing needs to be configured. Install `gh-pages` package:
   ```bash
   cd client
   npm install --save-dev gh-pages
   ```
   
   Update `client/package.json`:
   ```json
   {
     "homepage": "https://yourusername.github.io/email_website",
     "scripts": {
       "predeploy": "npm run build",
       "deploy": "gh-pages -d build"
     }
   }
   ```
   
   Then deploy:
   ```bash
   cd client
   npm run deploy
   ```

### Option B: Deploy Backend + Frontend (Heroku, Render, Railway)

1. **Build frontend:**
   ```bash
   cd client && npm run build && cd ..
   ```

2. **Deploy to platform (example: Railway):**
   - Connect your GitHub repo
   - Select Node.js environment
   - Railway auto-runs `npm start` (serves frontend)
   - Your app is now live!

## How It Works

### Data Flow

```
User Creates Survey
    ↓
surveyStorage.createSurvey() [utils/surveyStorage.js]
    ↓
Data saved to localStorage
    ↓
Redux state updated
    ↓
Components re-render with new data
```

### surveyStorage API

Located in `client/src/utils/surveyStorage.js`:

```javascript
surveyStorage.getAllSurveys()           // Get all surveys
surveyStorage.getSurveyById(id)         // Get one survey
surveyStorage.createSurvey(...)         // Create new survey
surveyStorage.updateSurveyVotes(id, choice)  // Update yes/no counts
surveyStorage.markRecipientResponded()  // Mark email as responded
surveyStorage.clear()                   // Clear all data
```

## Limitations

1. **Data is local to browser:** Each user has their own survey list
2. **No email sending:** Surveys can't be sent to recipients automatically
3. **No voting via email:** Recipients can't click email links to vote
4. **Data lost on clear:** Browser cache clear = data gone
5. **No user accounts:** All surveys are anonymous

## Troubleshooting

### Surveys disappear after refresh?
- Browser localStorage might be disabled
- Try incognito mode (creates fresh localStorage)
- Check DevTools → Application → Storage

### Strange behavior in production?
- Make sure `client/build` is committed and up-to-date
- Clear browser cache and rebuild
- Check that API paths don't exist (we removed them)

### Want email functionality back?
- We removed Mailer service, but you can:
  - Deploy backend to a separate service (Vercel, Railway)
  - Add SendGrid integration back to `services/Mailer.js`
  - Update `client/src/actions/index.js` to call API endpoints
